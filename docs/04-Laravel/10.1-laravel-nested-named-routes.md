# Route-ok, named route-ok és nested resource

Ebben a leckében a Laravel route rendszerét vizsgáljuk meg részletesen, különös tekintettel:

- a `Route::resource` és `Route::apiResource` használatára
- a named route-okra
- a nested resource struktúrára
- a `shallow()` megoldásra

A példákban a **counties–cities** domaint használjuk (1:N kapcsolat).

---

## 1. Resource route alapok

### County resource

```php
Route::resource('counties', CountyController::class);
```

Ez automatikusan létrehozza a teljes CRUD route készletet:

| Method    | URI                     | Action  |
| --------- | ----------------------- | ------- |
| GET       | `/counties`               | index   |
| GET       | `/counties/create`        | create  |
| POST      | `/counties`               | store   |
| GET       | `/counties/{county}`      | show    |
| GET       | `/counties/{county}/edit` | edit    |
| PUT/PATCH | `/counties/{county}`      | update  |
| DELETE    | `/counties/{county}`      | destroy |

---

## 2. Nested resource – counties és cities

```php
Route::resource('counties.cities', CityController::class);
```

Ez az alábbi route-okat hozza létre:

| Method    | URI                                   |
| --------- | ------------------------------------- |
| GET       | `/counties/{county}/cities`             |
| GET       | `/counties/{county}/cities/create`      |
| POST      | `/counties/{county}/cities`             |
| GET       | `/counties/{county}/cities/{city}`      |
| GET       | `/counties/{county}/cities/{city}/edit` |
| PUT/PATCH | `/counties/{county}/cities/{city}`      |
| DELETE    | `/counties/{county}/cities/{city}`      |

---

## 3. Mikor indokolt a nested struktúra?

### Előnyei:

- egyértelműen látszik a kapcsolat (City → County)
- REST szemléletű
- jól dokumentálható
- kontextust ad a város műveleteihez

### Példa:

```
GET /counties/3/cities/create
```

---

## 4. A probléma: túl mély URL struktúra

A nested resource egyik hátránya:

- a `city edit`
- `city update`
- `city delete`

URL-ek nagyon hosszúak lesznek:

```
/counties/{county}/cities/{city}/edit
```

Sok esetben a `city_id` önmagában elég.

---

## 5. A megoldás: `->shallow()`

```php
Route::resource('counties.cities', CityController::class)
    ->shallow();
```

### Mit csinál a `shallow()`?

- A **create** és **store** route továbbra is nested marad
- De a **show, edit, update, destroy** már NEM tartalmazza a county azonosítót

---

### Shallow után a route-ok:

#### Nested marad:

```
/counties/{county}/cities
/counties/{county}/cities/create
/counties/{county}/cities (POST)
```

#### Egyszerűsített lesz:

```
/cities/{city}
```

```
/cities/{city}/edit
```

```
/cities/{city} (PUT/PATCH)
```

```
/cities/{city} (DELETE)
```

---

## 6. Miért jó a shallow?

- a város egyértelműen azonosítható `city_id` alapján
- a szerkesztéshez nem kell újra megadni a county_id-t
- tisztább URL struktúra
- kevesebb redundancia

---

## 7. Named route-ok

Példák shallow nested esetén:

```
counties.cities.index
counties.cities.create
cities.show
cities.edit
cities.update
cities.destroy
```

---

### Named route használat Blade-ben

#### Szerkesztés link:

```bladehtml
<a href="{{ route('cities.edit', $city->id) }}">
    Szerkesztés
</a>
```

#### Törlés gomb:

```bladehtml
<form action="{{ route('cities.destroy', $city->id) }}" method="POST">
    @csrf
    @method('DELETE')
    <button type="submit">Törlés</button>
</form>
```

---

## 8. Route model binding nested esetben

```php
public function index(County $county)
{
    $cities = $county->cities()->orderBy('name')->get();

    return view('cities.index', compact('county', 'cities'));
}
```

Laravel automatikusan:

- a ``{county}`` paraméterből `County` objektumot készít
- a ``{city}`` paraméterből `City` objektumot készít

---

## 9. Route::apiResource

```php
Route::apiResource('counties', CountyController::class);
```

Ez NEM hoz létre:

- create
- edit

route-okat.

Csak:

- index
- store
- show
- update
- destroy

Nested API példa:

```php
Route::apiResource('counties.cities', CityController::class)
    ->shallow();
```

---

## 10. Mikor használjunk melyiket?

### Web alkalmazás (Blade):

```
Route::resource()
```

### API:

```
Route::apiResource()
```

### 1:N kapcsolat esetén:

```
Route::resource('counties.cities', CityController::class)->shallow();
```
