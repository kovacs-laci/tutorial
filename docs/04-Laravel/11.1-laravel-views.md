# View-k és Blade fájlok

A **view-k** az MVC architektúra „V” komponensei.  
Feladatuk, hogy a controllerek által átadott adatokat megjelenítsék a felhasználónak.  
Laravelben a view-kat a **Blade sablonmotor** segítségével készítjük.

👉 Részletes dokumentáció: [Laravel Blade](https://laravel.com/docs/12.x/blade)  
👉 Blade direktívák: [Blade Directives](https://laravel.com/docs/12.x/blade#blade-directives)

---

## 1. Blade szintaxis alapok

- **Kiírás:** `{{ $valtozo }}` (HTML-escaped)
- **Raw kiírás:** `{!! $html !!}`
- **Feltétel:** `@if`, `@elseif`, `@else`, `@endif`
- **Ciklus:** `@foreach`, `@for`, `@while`
- **Include:** `@include('partial')`
- **Direktívák:**
    - `@csrf` → CSRF token beszúrása formokba
    - `@method('DELETE')` → HTTP metódus felülírás formban
    - `@error('field')` → validációs hiba kiírása
    - `@auth` / `@guest` → autentikáció alapú megjelenítés

## 2. Biztonság – CSRF védelem

### Mi az a CSRF?
- **CSRF** = *Cross‑Site Request Forgery* (Keresztoldali kérelem hamisítás).
- Ez egy támadási forma, amikor egy rosszindulatú weboldal megpróbál a felhasználó nevében műveleteket végrehajtani a te alkalmazásodban (pl. adat törlése, módosítása).

### Mire jó a `@csrf`?
- A Laravel minden **állapotváltoztató kéréshez** (POST, PUT, PATCH, DELETE) CSRF token ellenőrzést végez.
- A `@csrf` direktíva beszúrja ezt a tokent a formba:


<BladeTabs
plain={
  `<form method="POST" action="{{ route('counties.store') }}">
    @csrf
    <!-- mezők -->
  </form>`
}

bootstrap={
  `<form method="POST" action="{{ route('counties.store') }}" class="p-3 border rounded">
    @csrf
    <!-- mezők -->
  </form>`
}

tailwind={
  `<form method="POST" action="{{ route('counties.store') }}" class="p-4 bg-white shadow rounded">
    @csrf
    <!-- mezők -->
  </form>`
}
/>


- Ha hiányzik vagy nem egyezik → a kérés **403 Forbidden** hibával elutasításra kerül.

### 5.1 Lokalizáció

A navigációs résznézetben jól látszik, hogyan épül be a **lokalizáció** a Laravel alkalmazásba. A menüpontok feliratai nem közvetlenül a sablonban szerepelnek, hanem a `__('navigation.homepage')`, `__('navigation.counties')`, `__('navigation.cities')` és `__('navigation.logout')` hívásokon keresztül jelennek meg.

**Előnyök:**
- **Nyelvi fájlok használata:** A szövegek a `resources/lang/{locale}/navigation.php` fájlban definiálhatók, így könnyen karbantarthatók és bővíthetők.
- **Többnyelvű támogatás:** Az alkalmazás egyszerűen képes több nyelven megjeleníteni ugyanazt a felületet, a `locale` beállítás függvényében.
- **Egységesség:** A fordítási kulcsok (`navigation.homepage`, `navigation.logout` stb.) biztosítják, hogy mindenhol ugyanazt a kifejezést használjuk.
- **Rugalmas bővítés:** Új menüpont hozzáadásakor elég csak a kulcsot és a fordítást beírni a nyelvi fájlba.

---

**Példa nyelvi fájlok**

`resources/lang/hu/navigation.php`
```php
<?php

return [
    'homepage' => 'Kezdőlap',
    'counties'   => 'Megyék',
    'cities'   => 'Városok',
    'login'    => 'Bejelentkezés',
    'logout'   => 'Kijelentkezés',
```

`resources/lang/en/navigation.php`

```php
return [
```

A kulcsok (`homepage`, `counties`, `cities`, `login`, `logout`) felelnek meg a Blade sablonban használt `__('navigation.xxx')` hívásoknak. A fordítási értékek a beállított `locale` alapján jelennek meg.

### 5.2 A locale beállítása és módosítása

Laravelben a **locale** határozza meg, melyik nyelvi fájlt használja az alkalmazás.

- **Alapértelmezett beállítás:**  
  `config/app.php` fájlban:
  ```php
  'locale' => 'hu',
  ```
- **Módosítás futásidőben:**
  ```php
  use Illuminate\Support\Facades\App;

  App::setLocale('en');
  ```
- **Automatikus nyelvválasztás middleware segítségével:**
  ```php
  public function handle($request, Closure $next)
  {
      $locale = $request->get('lang', config('app.locale'));
      App::setLocale($locale);

      return $next($request);
  }
  ```

**Összefoglalva:**
- A fordítási kulcsok a nyelvi fájlokban vannak (`resources/lang/{locale}/...`).
- A `locale` értéke határozza meg, melyik nyelv aktív.
- Alapértelmezett nyelv a `config/app.php`‑ban állítható.
- Futásidőben `App::setLocale()` segítségével módosítható.
- Middleware‑ben vagy query paraméterrel dinamikusan is váltható.

---

**Példa URL-ek nyelvváltásra:**
- Magyar felület:
  ```
  http://localhost/movies?lang=hu
  ```
- Angol felület:
  ```
  http://localhost/movies?lang=en
  ```

**Várható eredmény:**
- A navigációs menü magyarul jelenik meg, ha `?lang=hu`.
- Angolul jelenik meg, ha `?lang=en`.

---

### 5.3 Middleware magyarázat

A middleware egy **közbenső réteg** a kliens kérése és a controller között. Olyan, mint egy szűrő vagy kapuőr: minden HTTP kérés átmegy rajta, mielőtt eljutna a tényleges logikához.

**Tipikus feladatok:** autentikáció ellenőrzése, jogosultságok vizsgálata, naplózás, nyelv beállítása.

**Locale beállítás middleware-ben:**
1. A felhasználó küld egy kérést, pl. `http://localhost/movies?lang=en`.
2. A middleware fut le a controller előtt.
3. Megnézi a query paramétert (`lang=en`).
4. Beállítja a `App::setLocale('en')` értéket.
5. A teljes kérés feldolgozása angol nyelvi fájlokkal történik.

**Példa middleware kód:**

```bash
php artisan make:middleware SetLocale
```

`app/Http/Middleware/SetLocale.php`
```php
<?php

namespace App\Http\Middleware;

use Closure;
use Illuminate\Support\Facades\App;

class SetLocale
{
    public function handle($request, Closure $next)
    {
        $locale = $request->get('lang', config('app.locale'));
        App::setLocale($locale);

        return $next($request);
    }
}
```

**Bekötés a Kernelben:**
```php
protected $middleware = [
    // ...
    \App\Http\Middleware\SetLocale::class,
];
```

**Összefoglalva:**
- A middleware egy előszoba, ahol minden kérés átmegy.
- Ebben tudjuk dinamikusan beállítani a `locale` értékét.
- Így a felhasználó egyszerűen tudja váltani a nyelvet az URL‑ben.

```php
    public function index(Request $request) {
        $query = County::query();

        $needle = trim($request->get('needle'));
        if ($needle) {
            $query->where('name', 'like', "%{$needle}%");
        }

        $entities = $query->orderBy('name')->get();

        return response()->json([
            'data' => $entities,
            'meta' => [
                'count' => $entities->count(),
            ]
        ]);
    }
```

# 13. Kapcsolatok megjelenítése – városok egy vármegyében

A `County` és `City` modellek között **1:N kapcsolat** van:

* egy vármegyének több városa lehet
* egy város egy vármegyéhez tartozik

### County modell

```php
public function cities()
{
    return $this->hasMany(City::class);
}
```

### City modell

```php
public function county()
{
    return $this->belongsTo(County::class);
}
```

---

## Controller módosítása

A `show()` metódusban érdemes **betölteni a városokat is**.

```php
public function show(County $county)
{
    $county->load('cities');

    return view('counties.show', [
        'entity' => $county
    ]);
}
```

Az `entity` változó tehát tartalmazza:

```
county
 └── cities (collection)
```

# 14. Lapozás (Pagination)

Ha sok rekord van az adatbázisban, a teljes lista egyszerre történő megjelenítése nem hatékony.
Laravel beépített **lapozást (pagination)** biztosít.

Dokumentáció:
[https://laravel.com/docs/12.x/pagination](https://laravel.com/docs/12.x/pagination)

---

## Controller módosítása

`CountyController@index`

```php
public function index(Request $request)
{
    $query = County::query();

    $needle = trim($request->get('needle'));

    if ($needle) {
        $query->where('name','like',"%{$needle}%");
    }

    $entities = $query
        ->orderBy('name')
        ->paginate(10);

    return view('counties.index', [
        'entities' => $entities
    ]);
}
```

---

### Mi változott?

Korábban:

```
get()
```

Most:

```
paginate(10)
```

Ez azt jelenti, hogy **egyszerre 10 rekord jelenik meg**.

---

## Blade módosítása

`resources/views/counties/index.blade.php`

A táblázat után jelenítsük meg a lapozó linkeket:

<BladeTabs
  plain={
    `{{ $entities->links() }}`
  }
  
  bootstrap={
    `{{-- Bootstrap 5 paginator --}}
    {{ $entities->links('pagination::bootstrap-5') }}`
  }
  
  tailwind={
    `{{-- Tailwind paginator (Laravel 8+) --}}
    {{ $entities->links() }}`
  }
/>
---

## Hogyan működik?

A Laravel automatikusan kezeli a query paramétert:

```
?page=1
?page=2
?page=3
```

Példa URL:

```
/counties?page=2
```

---

## Keresés + lapozás együtt

Ha keresést használunk, a lapozásnál meg kell tartani a query paramétereket.

Controller:

```php
$entities = $query
    ->orderBy('name')
    ->paginate(10)
    ->withQueryString();
```

Így a `needle` paraméter megmarad lapozáskor.

# 15. Összegzés

Ebben a leckében megismerkedtünk:

- A **Blade sablonmotorral**
- A **layout** (`layouts.app`) biztosítja az alap HTML keretet.
- Az **`make:view`** parancs gyorsan létrehozza a szükséges Blade fájlokat.
- A **résznézetek** (`navigation`, `toolbar`, `flash`) újrahasznosíthatók több oldalon.
- A **CRUD nézetek** (`index`, `show`, `create`, `edit`) teljes folyamatot lefednek.
- A **Blade direktívák** egyszerűsítik a formok és feltételek kezelését.
- A **CSRF védelem** (`@csrf`) alapvető biztonsági elem minden formban.
- A **lokalizáció** (`__()`) előkészíti a frontendet többnyelvű megjelenítésre.
- A **layout és résznézetek használatával**
- A **CSRF védelemmel**
- Az **Eloquent kapcsolatok** megjelenítésével
- A Laravel **pagination** használatával

Ezek együtt alkotják egy Laravel alkalmazás **nézet (view) rétegének alapjait**.
