# Policy és Nested Policy – Jogosultságkezelés

A **Policy** osztályok a Laravelben a jogosultsági logika központi helyei.  
Segítségükkel meghatározható, hogy egy felhasználó végrehajthat-e egy adott műveletet egy adott modellen.

Ez a lecke a **Counties–Cities** domain példáján mutatja be a működést.

---

## 1. Mi az a Policy?

A Policy egy osztály, amely egy modellhez tartozó jogosultsági szabályokat tartalmaz.

Példa:
- `CountyPolicy`
- `CityPolicy`

A policy metódusok visszatérési értéke:
- `true` → engedélyezett
- `false` → tiltott

---

## 2. Policy létrehozása

```bash
php artisan make:policy CountyPolicy --model=County
php artisan make:policy CityPolicy --model=City
````

A policy-k az `app/Policies` mappába kerülnek.

---

## 3. Policy regisztrálása

Laravel automatikusan felismeri a policy-kat,
de szükség esetén a `AuthServiceProvider` fájlban is regisztrálhatók.

---

## 4. Alap Policy példa – County

```php
public function update(User $user, County $county): bool
{
    return $user->id === $county->user_id;
}

public function delete(User $user, County $county): bool
{
    return $user->id === $county->user_id;
}
```

Ez azt jelenti:

* csak az a felhasználó módosíthatja vagy törölheti a megyét,
* aki létrehozta.

---

## 5. Policy használata controllerben

```php
public function edit(County $county)
{
    $this->authorize('update', $county);

    return view('counties.edit', compact('county'));
}
```

Ha a jogosultság nem teljesül:

* Laravel automatikusan `403 Forbidden` hibát ad.

---

## 6. Policy használata Blade-ben

### Szerkesztés link

```blade
@can('update', $county)
    <a href="{{ route('counties.edit', $county) }}">
        Szerkesztés
    </a>
@endcan
```

### Törlés gomb

```blade
@can('delete', $county)
<form method="POST" action="{{ route('counties.destroy', $county) }}">
    @csrf
    @method('DELETE')
    <button>Törlés</button>
</form>
@endcan
```

A `@can` direktíva a policy metódusokat hívja meg.

---

# Nested Policy

1:N kapcsolat esetén gyakran szükséges,
hogy az alárendelt modell jogosultsága a szülő modellen alapuljon.

Példánkban:

* Egy **County** több **City**-t tartalmaz.
* A `cities` táblában szerepel `county_id`.

---

## 7. Nested Create jogosultság

Route példa:

```
/counties/{county}/cities/create
```

Controller:

```php
public function create(County $county)
{
    $this->authorize('createCity', $county);

    return view('cities.create', compact('county'));
}
```

---

## 8. Nested Policy metódus

`CountyPolicy` vagy `CityPolicy`:

```php
public function createCity(User $user, County $county): bool
{
    return $user->id === $county->user_id;
}
```

Ez azt jelenti:

* Várost csak az hozhat létre,
* aki jogosult az adott megyére.

A jogosultság tehát a **szülő entitáson keresztül** történik.

---

## 9. Nested Store példa

```php
public function store(Request $request, County $county)
{
    $this->authorize('createCity', $county);

    $validated = $request->validate([
        'name' => 'required|string|max:255',
    ]);

    $validated['county_id'] = $county->id;

    City::create($validated);

    return redirect()->route('counties.show', $county);
}
```

Fontos:

* A `county_id` nem a formból jön,
* hanem a route paraméterből.
* Ez biztosítja az adatintegritást.

---

## 10. Nested Update jogosultság

```php
public function update(User $user, City $city): bool
{
    return $user->id === $city->county->user_id;
}
```

Itt a jogosultság a kapcsolaton keresztül ellenőrzött:

```
City → County → user_id
```

Ez a klasszikus **nested policy minta**.

---

## 11. AuthorizeResource használata

```php
public function __construct()
{
    $this->authorizeResource(City::class, 'city');
}
```

Ez automatikusan meghívja a megfelelő policy metódusokat
a resource controller műveleteinél.

---

## 12. Összefoglalás

* A **Policy** a jogosultsági logika helye.
* A controller az `authorize()` metódussal ellenőriz.
* A Blade az `@can` direktívával jeleníti meg a jogosultság alapján az elemeket.
* Nested Policy esetén a jogosultság a szülő modellen keresztül történik.
* 1:N kapcsolatoknál ez a javasolt megközelítés.
