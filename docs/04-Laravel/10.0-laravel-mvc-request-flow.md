# Laravel MVC és a kérés feldolgozása

Ebben a leckében áttekintjük, hogyan kapcsolódik össze a **Route**, a **Controller** és a **View** a Laravelben, és hogyan dolgoz fel a rendszer egy HTTP kérést az MVC architektúra mentén.

A példákban a **counties–cities** domaint használjuk.

---

## 1. Az MVC röviden

Az **MVC (Model–View–Controller)** egy tervezési minta, amely szétválasztja az alkalmazás felelősségi köreit:

- **Model (M)** → adatok és adatbázis-műveletek
- **View (V)** → megjelenítés (Blade sablonok)
- **Controller (C)** → a kérés feldolgozása és irányítása

---

## 2. Az MVC folyamat – lépésről lépésre

Egy tipikus kérés feldolgozása így történik:

### 1️⃣ A felhasználó meghív egy URL-t

Például:

```

/counties

```

vagy

```

/counties/3/cities

````

---

### 2️⃣ A Route eldönti, melyik controller fut le

A route-ok a `routes/web.php` fájlban találhatók.

Példa:

```php
Route::resource('counties', CountyController::class);
````

Ez összekapcsolja az URL-eket a megfelelő controller metódusokkal.

---

### 3️⃣ A Controller feldolgozza a kérést

Példa:

```php
public function index()
{
    $counties = County::orderBy('name')->get();

    return view('counties.index', compact('counties'));
}
```

A controller feladata:

* adat lekérése a modelből
* üzleti logika végrehajtása
* adat továbbítása a view-nak

---

### 4️⃣ A Model kezeli az adatbázist

Példa:

```php
$counties = County::orderBy('name')->get();
```

Az **Eloquent ORM** segítségével a model objektum-orientált módon kommunikál az adatbázissal.

---

### 5️⃣ A View megjeleníti az adatokat

A controller ezt adja vissza:

```php
return view('counties.index', compact('counties'));
```

A `resources/views/counties/index.blade.php` fájl jeleníti meg az adatokat.

Példa Blade kóddal:

```blade
<h1>Megyék listája</h1>

<ul>
@foreach($counties as $county)
    <li>{{ $county->name }}</li>
@endforeach
</ul>
```

## 4. Hogyan illeszkedik ide a counties–cities példa?

### Példa: városok listázása egy megyéhez

Route:

```
/counties/3/cities
```

Controller:

```php
public function index(County $county)
{
    $cities = $county->cities()->orderBy('name')->get();

    return view('cities.index', compact('county', 'cities'));
}
```

Itt történik:

* Route Model Binding → a `{county}` automatikusan `County` objektummá alakul
* A controller lekéri a kapcsolódó városokat
* A view megjeleníti az adatokat

---

## 5. Miért fontos megérteni ezt az összefüggést?

Mielőtt a következő témákra lépnénk:

* route részletek
* named route-ok
* nested resource
* shallow routing
* lapozás
* keresés

Fontos tisztán látni, hogy:

* a route csak irányít
* a controller koordinál
* a model adatot kezel
* a view megjelenít

Ez az alap, amelyre a teljes Laravel alkalmazás épül.

## Összegzés

* A Laravel az MVC architektúrát követi.
* A kérés a route-on keresztül jut el a controllerhez.
* A controller a model segítségével dolgozik az adatokkal.
* A view jeleníti meg az eredményt.
* A counties–cities példa jól szemlélteti az 1:N kapcsolat kezelését az MVC-ben.
