# N:N kapcsolat Laravelben

Az **N:N (many-to-many)** kapcsolat azt jelenti, hogy:

- egy **tanár** több **tantárgyat** taníthat,
- egy **tantárgyat** több **tanár** is taníthat.

Ez a kapcsolat egy **kapcsolótáblán (pivot table)** keresztül valósul meg.

Ebben a példában:

- **Teacher** → Tanár
- **Subject** → Tantárgy
- **subject_teacher** → kapcsolótábla

---

## 1. Adatbázis struktúra

### `teachers` tábla

```php
Schema::create('teachers', function (Blueprint $table) {
    $table->id();
    $table->string('name');
});
````

### `subjects` tábla

```php
Schema::create('subjects', function (Blueprint $table) {
    $table->id();
    $table->string('name');
});
```

### Kapcsolótábla: `subject_teacher`

A Laravel konvenció szerint a tábla neve **ábécérendben**, singular formában:

```
subject_teacher
```

```php
Schema::create('subject_teacher', function (Blueprint $table) {
    $table->foreignId('teacher_id')
          ->constrained()
          ->onDelete('cascade');

    $table->foreignId('subject_id')
          ->constrained()
          ->onDelete('cascade');
});
```

### Fontos elemek

* nincs saját `id` mező
* csak két idegen kulcs
* `onDelete('cascade')` → kapcsolatok automatikus törlése

---

## 2. Modellek közötti kapcsolat

### Teacher modell

```php
namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Teacher extends Model
{
    public $timestamps = false;

    protected $fillable = ['name'];

    public function subjects()
    {
        return $this->belongsToMany(Subject::class);
    }
}
```

### Subject modell

```php
namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Subject extends Model
{
    public $timestamps = false;

    protected $fillable = ['name'];

    public function teachers()
    {
        return $this->belongsToMany(Teacher::class);
    }
}
```

### Kapcsolat típusa

* `belongsToMany()` → N:N kapcsolat
* Laravel automatikusan a `subject_teacher` táblát használja

Ha eltérő nevet használnánk, meg kell adni:

```php
return $this->belongsToMany(Subject::class, 'custom_table_name');
```

---

## 3. Kapcsolatok használata

### Tantárgyak lekérése egy tanárhoz

```php
$teacher = Teacher::find($id);
$subjects = $teacher->subjects;
```

### Tanárok lekérése egy tantárgyhoz

```php
$subject = Subject::find($id);
$teachers = $subject->teachers;
```

---

## 4. Kapcsolatok létrehozása

### 1. `attach()`

```php
$teacher = Teacher::find(1);
$teacher->subjects()->attach(2);
```

Ez összekapcsolja a tanárt a 2-es tantárggyal.

---

### 2. Több kapcsolat egyszerre

```php
$teacher->subjects()->attach([1, 2, 3]);
```

---

### 3. `sync()` (ajánlott)

A `sync()` frissíti a kapcsolatot:

```php
$teacher->subjects()->sync([1, 3]);
```

Ez azt jelenti:

* csak az 1 és 3 tantárgy marad meg
* minden más kapcsolat törlődik

---

### 4. `detach()`

```php
$teacher->subjects()->detach(2);
```

---

## 5. Eager Loading N:N kapcsolatnál

```php
$teachers = Teacher::with('subjects')->get();
```

vagy fordítva:

```php
$subjects = Subject::with('teachers')->get();
```

Ez ugyanúgy elkerüli az N+1 problémát.

---

## 6. Kapcsolótábla extra mezőkkel (opcionális)

A pivot tábla tartalmazhat további adatokat is.

### Példa: tanítási év

```php
Schema::create('subject_teacher', function (Blueprint $table) {
    $table->foreignId('teacher_id')->constrained()->onDelete('cascade');
    $table->foreignId('subject_id')->constrained()->onDelete('cascade');
    $table->string('school_year');
});
```

### Modellben:

```php
public function subjects()
{
    return $this->belongsToMany(Subject::class)
                ->withPivot('school_year');
}
```

### Használat:

```php
$teacher->subjects()->attach(1, ['school_year' => '2024-25']);
```

### Lekérés:

```php
foreach ($teacher->subjects as $subject) {
    echo $subject->pivot->school_year;
}
```

---

## 7. Controller példa

```php
class TeacherController extends Controller
{
    public function index()
    {
        $teachers = Teacher::with('subjects')->get();

        return view('teachers.index', compact('teachers'));
    }
}
```

---

## 8. Összegzés

* Az N:N kapcsolat külön **pivot táblát** igényel.
* A kapcsolat típusa: `belongsToMany()`.
* A pivot tábla:

    * nem tartalmaz `id` mezőt
    * két idegen kulcsot tartalmaz
* Kapcsolatkezelés:

    * `attach()`
    * `detach()`
    * `sync()`
* Eager loading használható: `with('subjects')`
* A pivot táblában extra mezők is tárolhatók a `withPivot()` segítségével.
