# Szülő–gyerek kapcsolat Laravelben

A szülő–gyerek kapcsolat (pl. *Gyártó → Modellek*) az egyik leggyakoribb reláció az adatbázisokban. Laravelben ezt migrációkban, modellekben és kontrollerekben is tisztán és egyszerűen lehet kezelni. Az alábbi összefoglaló végigvezet a teljes folyamaton.

---

## 1. Adatbázis és migrációk

A gyerek tábla idegen kulcsa mindig ugyanazt a típust használja, mint a szülő tábla elsődleges kulcsa.

### Példa: `models` tábla migrációja

`database/migrations/xxxx_xx_xx_create_models_table.php`

```php
Schema::create('models', function (Blueprint $table) {
    $table->id();
    $table->unsignedBigInteger('maker_id');
    $table->foreign('maker_id')->references('id')->on('makers')->onDelete('cascade');
    $table->string('name')->index();
});
```

vagy használhatunk migrációt egysoros idegen kulccsal is:

```php
Schema::create('models', function (Blueprint $table) {
    $table->id();
    $table->foreignId('maker_id')->constrained('makers')->onDelete('cascade');
    $table->string('name')->index();
});
```

**Lényeges elemek:**
- `maker_id` az idegen kulcs.
- `onDelete('cascade')` gondoskodik róla, hogy a gyártó törlésekor a modellek is törlődjenek.

## 2. Modellek közötti kapcsolat

A kapcsolatot a modellekben definiáljuk.

### Maker modell

```php
class Maker extends Model
{
    public $timestamps = false;

    protected $fillable = ['name'];

    public function models()
    {
        return $this->hasMany(CarModel::class);
    }
}
```

### CarModel modell

```php
class CarModel extends Model
{
    public $timestamps = false;

    protected $fillable = ['maker_id', 'name'];

    public function maker()
    {
        return $this->belongsTo(Maker::class);
    }
}
```

**Összefoglalva:**
- A szülő (`Maker`) `hasMany` kapcsolatot használ.
- A gyerek (`CarModel`) `belongsTo` kapcsolatot használ.

## 3. Kapcsolatok lekérése a kontrollerben (Eager Loading)

A kapcsolatok hatékony lekérdezéséhez a `with()` metódust használjuk. Ez előre betölti a kapcsolódó adatokat, így elkerülhető a N+1 lekérdezés probléma.

### Szülő → Gyerek lekérése

```php
$makers = Maker::with('models')->get();
```

pl. az index() metódusban:

**Szülő oldalon:**

```php
class MakerController extends Controller
{
    public function index()
    {
        // Összes Gyártó listázása az egyes gyártóhoz tartozó modellekkel együtt.
        $makers = Maker::with('models')->get();
        return view('makers.index', compact('makers'));
        // API esetén: return response()->json($makers);
    }
    
    (...)
}
```

### Gyerek → Szülő lekérése

```php
$models = CarModel::with('maker')->get();
```

pl. a gyerek kontrollerben:

```php
class ModelController extends Controller
{
    public function index()
    {
        // Összes modell listázása az egyes modellek gyártóival együtt
        $models = Model::with('vehicle')->get();
        return view('models.index', compact('models'));
        // API esetén: return response()->json($models);
    }
    
    (...)
}


**Mi történik a háttérben?**
- Laravel két lekérdezést futtat: egyet a szülőkre, egyet a gyerekekre.
- A kapcsolatot memóriában párosítja össze.
- A nézetben már nem generálódnak új SQL lekérdezések.

## 4. Feltételes (szűrt) eager loading

A `with()` callbacket is fogad, így a kapcsolatokra külön feltételeket tehetsz.

### Csak aktív modellek betöltése

```php
$makers = Maker::with([
    'models' => function ($query) {
        $query->where('active', true);
    }
])->get();
```

### Csak bizonyos mezők betöltése

```php
$makers = Maker::with([
    'models' => function ($query) {
        $query->select('id', 'maker_id', 'name');
    }
])->get();
```

### Limitálás és rendezés

```php
$makers = Maker::with([
    'models' => function ($query) {
        $query->orderBy('created_at', 'desc')->limit(5);
    }
])->get();
```

### Többszintű feltételes betöltés

```php
$makers = Maker::with([
    'models.vehicle' => function ($query) {
        $query->where('type', 'electric');
    }
])->get();
```

## 5. Kapcsolatok használata a kódban

### Szülő lekérése a gyerekből

```php
$model = CarModel::find($id);
$maker = $model->maker;
```

### Gyerekek lekérése a szülőből

```php
$maker = Maker::find($id);
$models = $maker->models;
```

## 6. Szülő–gyerek select mezők

Gyakori feladat, hogy a felhasználó először gyártót választ, majd ennek megfelelően jelennek meg a modellek.

### Route

```php
Route::get('/makers/{maker}/fetch-models', [MakerController::class, 'fetchModels'])
    ->name('makers.fetch.models');
```

### Controller metódus

```php
public function fetchModels($makerId)
{
    $maker = Maker::findOrFail($makerId);

    $result['data'] = $maker->models()->select('id', 'name')->get();

    return response()->json($result);
}
```

---

### 1. jQuery frontend (Ajax)

```javascript
$('#select-maker').change(function () {
    var id = $(this).val();
    $('#select-model').find('option').not(':first').remove();

    $.ajax({
        url: '/makers/' + id + '/fetch-models',
        type: 'get',
        dataType: 'json',
        success: function (response) {
            if (response.data) {
                response.data.forEach(function(model) {
                    $("#select-model").append(
                        "<option value='" + model.id + "'>" + model.name + "</option>"
                    );
                });
            }
        }
    });
});
```

---

### 2. Fetch API frontend (modern, jQuery nélkül)

```javascript
document.getElementById('select-maker').addEventListener('change', function () {
    const id = this.value;

    const modelSelect = document.getElementById('select-model');
    modelSelect.querySelectorAll('option:not(:first-child)').forEach(o => o.remove());

    fetch(`/makers/${id}/fetch-models`)
        .then(response => response.json())
        .then(data => {
            if (data.data) {
                data.data.forEach(model => {
                    const option = document.createElement('option');
                    option.value = model.id;
                    option.textContent = model.name;
                    modelSelect.appendChild(option);
                });
            }
        })
        .catch(error => console.error('Hiba:', error));
});
```

**Jellemzők:**
- nincs szükség jQuery‑re,
- egyszerű, natív böngésző API,
- jól olvasható, rövid kód.

---

### 3. Async/await frontend (még tisztább, ajánlott)

```javascript
document.getElementById('select-maker').addEventListener('change', async function () {
    const id = this.value;

    const modelSelect = document.getElementById('select-model');
    modelSelect.querySelectorAll('option:not(:first-child)').forEach(o => o.remove());

    try {
        const response = await fetch(`/makers/${id}/fetch-models`);
        const data = await response.json();

        if (data.data) {
            data.data.forEach(model => {
                const option = document.createElement('option');
                option.value = model.id;
                option.textContent = model.name;
                modelSelect.appendChild(option);
            });
        }
    } catch (error) {
        console.error('Hiba:', error);
    }
});
```

**Előnyök:**
- a kód szinkron jellegűnek tűnik,
- könnyebb hibakezelés,
- jól skálázható, ha később több API‑hívás lesz.

## 7. Összegzés

- A migrációban az idegen kulcs típusa egyezzen a szülő kulcsával.
- A modellekben `hasMany` és `belongsTo` kapcsolatokkal építjük fel a relációt.
- A `with()` metódus előre betölti a kapcsolódó adatokat, így gyorsabb és hatékonyabb a lekérdezés.
- A feltételes eager loading segítségével szűrheted, rendezheted vagy limitálhatod a kapcsolódó rekordokat.
- AJAX segítségével dinamikus szülő–gyerek select mezők hozhatók létre.
