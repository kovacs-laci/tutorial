# HTTP tesztelés – autentikáció és jogosultság

Ez a lecke a **HTTP feature tesztek** használatát mutatja be Laravelben,
különös tekintettel az **autentikációra** és az **authorizationre (jogosultságkezelésre)**.

A példák a **counties–cities** domainen alapulnak.

---

## 1. Tesztkörnyezet

### In-memory adatbázis

`phpunit.xml`:

```xml
<php>
    <env name="DB_CONNECTION" value="sqlite"/>
    <env name="DB_DATABASE" value=":memory:"/>
</php>
````

Ez biztosítja, hogy a tesztek ne módosítsák a valódi adatbázist.

---

## 2. Szükséges trait

Minden HTTP tesztnél ajánlott:

```php
use Illuminate\Foundation\Testing\RefreshDatabase;
```

Ez minden teszt előtt újratölti az adatbázist.

---

## 3. Autentikáció tesztelése

Laravelben a tesztekben a `actingAs()` metódust használjuk.

### Példa: bejelentkezett felhasználó

```php
$this->actingAs(User::factory()->create());
```

---

## 4. County CRUD – autentikációval

Tegyük fel, hogy:

* a **listázás** publikus
* a **létrehozás, módosítás, törlés** csak bejelentkezett felhasználónak engedélyezett

---

### 4.1 Lista teszt

```php
public function test_anyone_can_view_counties_index()
{
    County::factory()->count(3)->create();

    $response = $this->get(route('counties.index'));

    $response->assertStatus(200);
}
```

---

### 4.2 Létrehozás – autentikált felhasználó

```php
public function test_authenticated_user_can_create_county()
{
    $this->actingAs(User::factory()->create());

    $data = ['name' => 'Test County'];

    $response = $this->post(route('counties.store'), $data);

    $response->assertRedirect(route('counties.index'));
    $this->assertDatabaseHas('counties', $data);
}
```

---

### 4.3 Létrehozás – vendég felhasználó

```php
public function test_guest_cannot_create_county()
{
    $data = ['name' => 'Test County'];

    $response = $this->post(route('counties.store'), $data);

    $response->assertRedirect(route('login'));
}
```

Ez akkor működik így, ha a `auth` middleware védi az útvonalat.

---

## 5. City CRUD – autentikációval

Ugyanez érvényes a `cities` erőforrásra is.

---

### 5.1 Város létrehozása

```php
public function test_authenticated_user_can_create_city()
{
    $this->actingAs(User::factory()->create());

    $county = County::factory()->create();

    $data = [
        'name' => 'Test City',
        'county_id' => $county->id,
    ];

    $response = $this->post(route('cities.store'), $data);

    $response->assertRedirect(route('cities.index'));
    $this->assertDatabaseHas('cities', $data);
}
```

---

### 5.2 Törlés

```php
public function test_authenticated_user_can_delete_city()
{
    $this->actingAs(User::factory()->create());

    $city = City::factory()->create();

    $response = $this->delete(route('cities.destroy', $city->id));

    $response->assertRedirect(route('cities.index'));
    $this->assertDatabaseMissing('cities', ['id' => $city->id]);
}
```

---

## 6. Authorization tesztelés (Policy)

Az authorization célja annak ellenőrzése, hogy a felhasználó **rendelkezik-e jogosultsággal** egy adott művelethez.

Laravelben ezt leggyakrabban **Policy** segítségével kezeljük.

Például:

* csak admin törölhet várost
* csak az adott megyéhez tartozó felhasználó szerkeszthet

---

## 7. Policy teszt példa

Tegyük fel, hogy van `CityPolicy`, és a törléshez admin szerepkör szükséges.

### 7.1 Engedélyezett művelet

```php
public function test_admin_can_delete_city()
{
    $admin = User::factory()->create([
        'role' => 'admin',
    ]);

    $this->actingAs($admin);

    $city = City::factory()->create();

    $response = $this->delete(route('cities.destroy', $city->id));

    $response->assertRedirect(route('cities.index'));
    $this->assertDatabaseMissing('cities', ['id' => $city->id]);
}
```

---

### 7.2 Tiltott művelet

```php
public function test_non_admin_cannot_delete_city()
{
    $user = User::factory()->create([
        'role' => 'user',
    ]);

    $this->actingAs($user);

    $city = City::factory()->create();

    $response = $this->delete(route('cities.destroy', $city->id));

    $response->assertStatus(403);
}
```

A `403 Forbidden` státuszkód jelzi, hogy a Policy elutasította a műveletet.

---

## 8. HTTP státuszkód ellenőrzések

Tesztelés során gyakran használjuk:

```php
$response->assertStatus(200);
$response->assertStatus(302);
$response->assertStatus(403);
$response->assertRedirect();
```

---

## 9. Middleware tesztelés

Ha az útvonal `auth` middleware-rel védett:

```php
$response = $this->get(route('cities.create'));
$response->assertRedirect(route('login'));
```

Ez biztosítja, hogy vendég nem fér hozzá.

public function definition(): array
{

        'name' => $this->faker->state,
    ];

```

    ];
}
```

## 12. Összegzés

* A HTTP feature tesztek a teljes útvonalat vizsgálják.
* Az autentikáció teszteléséhez `actingAs()` használható.
* A vendég hozzáférés ellenőrizhető `assertRedirect()`-tel.
* Az authorization (Policy) tesztek `403` státuszkódot várnak jogosulatlan esetben.
* A counties–cities domain jól használható CRUD + jogosultság tesztek bemutatására.
* A `RefreshDatabase` és Factory használata biztosítja az izolált tesztkörnyezetet.
