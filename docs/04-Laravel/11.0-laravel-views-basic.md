# View-k és Blade fájlok

A **view-k** az MVC architektúra „V” komponensei.  
Feladatuk, hogy a controller által átadott adatokat megjelenítsék a felhasználónak.

Laravelben a view-kat a **Blade sablonmotor** segítségével készítjük.

👉 Dokumentáció: https://laravel.com/docs/12.x/blade

---

## 1. A `resources/views` mappa szerkezete

A view fájlok a következő helyen találhatók:

```

resources/views

```

Ajánlott struktúra counties–cities példával:

```

resources/views/
│
├── layouts/
│   └── app.blade.php
│
├── counties/
│   ├── index.blade.php
│   ├── show.blade.php
│   ├── create.blade.php
│   └── edit.blade.php
│
└── cities/
    ├── index.blade.php
    ├── show.blade.php
    ├── create.blade.php
    └── edit.blade.php


````

Ez a struktúra a modellekhez igazodik.

---

## 2. View létrehozása Artisan paranccsal

Új view fájl létrehozása:

```bash
php artisan make:view counties.index
````

Ez létrehozza:

```
resources/views/counties/index.blade.php
```

> Ha a `make:view` parancs nem elérhető, a fájl manuálisan is létrehozható.

---

## 3. View visszaadása a Controllerből

Példa:

```php
public function index()
{
    $counties = County::all();

    return view('counties.index', compact('counties'));
}
```

A `view()` függvény első paramétere:

```
mappa.fájlnév
```

A második paraméter az adatok átadása.

---

## 4. Blade szintaxis alapok

### Kiírás

```bladehtml
{{ $county->name }}
```

### Feltétel

```bladehtml
@if($counties->isEmpty())
    Nincs adat.
@endif
```

### Ciklus

```bladehtml
@foreach($counties as $county)
    <p>{{ $county->name }}</p>
@endforeach
```

---

## 5. Layout használata

A layout egy központi sablon, amely tartalmazza:

* HTML szerkezet
* navigáció
* lábléc
* közös elemek
* és egy tartalom helyet

### Layout fájl:

```
resources/views/layouts/app.blade.php
```

### Tartalom helye a layoutban:

```bladehtml
@yield('content')
```

---

### Oldal a layout használatával

```bladehtml
@extends('layouts.app')

@section('content')

    <h1>Vármegyék</h1>

    @foreach($counties as $county)
        <p>{{ $county->name }}</p>
    @endforeach

@endsection
```

---

## 6. Résznézetek használata

Az ismétlődő elemek külön fájlba szervezhetők.

### Példák:

```
resources/views/layouts/navigation.blade.php
resources/views/layouts/footer.blade.php
resources/views/layouts/search-form.blade.php
```

Beszúrás:

```bladehtml
@include('layouts.navigation')
@include('layouts.footer')
@include('layouts.search-form')
```

## 7. Keresés a listázó oldalon

### Controller (counties):

```php
public function index(Request $request)
{
    $query = County::query();

    if ($request->filled('needle')) {
        $query->where('name', 'like', '%' . $request->needle . '%');
    }

    $counties = $query
        ->orderBy('name')
        ->paginate(10);

    return view('counties.index', compact('counties'));
}
```

---

### View – kereső form

```bladehtml
<form method="GET" action="{{ route('counties.index') }}">
    <input type="search" name="needle" value="{{ request('needle') }}">
    <button type="submit">Keresés</button>
</form>
```

---

## 8. Lapozás (Pagination)

A `paginate()` metódus automatikusan lapozható eredményt ad.

A view-ban:

```bladehtml
<div class="paginator">
    {{ $counties->links() }}
</div>
```

Ez generálja az oldalváltó linkeket.

Fontos:
A lapozás csak akkor működik, ha a controllerben `paginate()` szerepel, nem pedig `get()`.

---

## 9. Named route használata linkekben

A view-ban mindig ajánlott a route nevére hivatkozni.

### Példa:

```bladehtml
<a href="{{ route('counties.edit', $county->id) }}">
    Szerkesztés
</a>
```

Törlés gomb:

```bladehtml
<form method="POST" action="{{ route('counties.destroy', $county->id) }}">
    @csrf
    @method('DELETE')
    <button type="submit">Törlés</button>
</form>
```

Ez stabil megoldás, mert az URL változhat, de a route név nem.

---

## 10. Összefoglalás

* A view-k a `resources/views` mappában találhatók.
* A `php artisan make:view` paranccsal hozhatók létre.
* A controller a `view()` függvénnyel adja vissza őket.
* A Blade sablonmotor kezeli a megjelenítést.
* A layout biztosítja az oldal szerkezetét.
* A résznézetek újrahasznosítható elemek.
* A keresés `Request` alapján történik.
* A lapozás a `paginate()` metódussal működik.
* A linkekben ajánlott named route-ot használni.
