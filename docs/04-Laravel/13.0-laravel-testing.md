# Automata tesztelés Laravelben

A Laravel beépítve tartalmazza a PHPUnit tesztkeretrendszert, így nincs szükség külön telepítésre.  
Ha mégis manuálisan szeretnénk telepíteni, a következő parancs használható:

```bash
composer require --dev phpunit/phpunit
````

Dokumentáció:
[https://laravel.com/docs/11.x/testing](https://laravel.com/docs/11.x/testing)

---

# 1. Adatbázis beállítás tesztekhez

A `phpunit.xml` fájlban módosítsuk a `<php>` szekciót, hogy a tesztek **memóriában futó SQLite adatbázist** használjanak.

Így a tesztek futása során a valódi adatbázis **nem módosul**.

```xml
<php>
    <env name="DB_CONNECTION" value="sqlite"/>
    <env name="DB_DATABASE" value=":memory:"/>
</php>
```

# 2. Teszt létrehozása

### Feature teszt

```bash
php artisan make:test CountyTest
```

### Unit teszt

```bash
php artisan make:test CountyTest --unit
```

Fontos: a teszt neve mindig `Test` végződésű legyen.

---

# 3. RefreshDatabase trait

A tesztosztályban használjuk a `RefreshDatabase` traitet, hogy minden teszt **tiszta adatbázissal induljon**.

```php
use Illuminate\Foundation\Testing\RefreshDatabase;

class CountyTest extends TestCase
{
    use RefreshDatabase;
}
```

A trait minden teszt előtt újramigrálja az adatbázist.

# 4. Factory használata

Teszteket általában **tesztadatokkal** írunk.
Ehhez Laravel Factory osztályokat használ.

Factory létrehozása:

```bash
php artisan make:factory CountyFactory
```

`database/factories/CountyFactory.php`

```php
namespace Database\Factories;

use App\Models\County;
use Illuminate\Database\Eloquent\Factories\Factory;

class CountyFactory extends Factory
{
    protected $model = County::class;

    public function definition(): array
    {
        return [
            'name' => $this->faker->city,
        ];
    }
}
```

A Faker számos adattípust képes generálni:

* név
* email
* cím
* dátum
* telefonszám

# 5. Feature tesztek vs. Unit tesztek

## Feature tesztek

A feature tesztek egy teljes funkció működését vizsgálják.

Mit tesztelnek?

```
Route → Controller → Model → View
```

Előnyük:

* közel állnak a valós használathoz

Hátrányuk:

* lassabbak lehetnek

Példa:

```php
public function test_user_can_create_county()
{
    $this->actingAs(User::factory()->create());

    $response = $this->post(route('counties.store'), [
        'name' => 'Test County'
    ]);

    $response->assertRedirect(route('counties.index'));

    $this->assertDatabaseHas('counties', [
        'name' => 'Test County'
    ]);
}
```

---

## Unit tesztek

A unit tesztek egyetlen osztály vagy metódus működését vizsgálják.

Előnyük:

* gyorsak
* izoláltan futnak

Hátrányuk:

* nem vizsgálják a teljes rendszert

Példa egy Eloquent kapcsolatra:

```php
public function test_county_has_cities()
{
    $county = County::factory()
        ->hasCities(3)
        ->create();

    $this->assertEquals(
        3,
        $county->cities()->count()
    );
}
```

# 6. CRUD tesztek

`tests/Feature/CountyControllerTest.php`

```php
use App\Models\County;
use App\Models\User;
use Illuminate\Foundation\Testing\RefreshDatabase;

class CountyControllerTest extends TestCase
{
    use RefreshDatabase;

    /** Listázás */
    public function test_user_can_view_counties_index()
    {
        County::factory()->count(3)->create();

        $response = $this->get(route('counties.index'));

        $response->assertStatus(200);
        $response->assertViewHas('entities');
    }

    /** Létrehozás */
    public function test_authenticated_user_can_create_county()
    {
        $this->actingAs(User::factory()->create());

        $data = [
            'name' => 'Test County'
        ];

        $response = $this->post(
            route('counties.store'),
            $data
        );

        $response->assertRedirect(route('counties.index'));

        $this->assertDatabaseHas('counties', $data);
    }

    /** Módosítás */
    public function test_authenticated_user_can_update_county()
    {
        $county = County::factory()->create();

        $this->actingAs(User::factory()->create());

        $updatedData = [
            'name' => 'Updated County'
        ];

        $response = $this->patch(
            route('counties.update', $county->id),
            $updatedData
        );

        $this->assertDatabaseHas('counties', $updatedData);

        $response->assertRedirect(route('counties.index'));
    }

    /** Törlés */
    public function test_authenticated_user_can_delete_county()
    {
        $county = County::factory()->create();

        $this->actingAs(User::factory()->create());

        $response = $this->delete(
            route('counties.destroy', $county->id)
        );

        $this->assertDatabaseMissing('counties', [
            'id' => $county->id
        ]);

        $response->assertRedirect(route('counties.index'));
    }
}
```

# 7. Tesztek futtatása

Az összes teszt futtatása:

```bash
vendor/bin/phpunit
```

Csak egy adott tesztosztály futtatása:

```bash
vendor/bin/phpunit --filter CountyTest
```

Csak egy tesztmetódus futtatása:

```bash
vendor/bin/phpunit --filter CountyTest::test_user_can_view_counties_index
```

Formázott kimenet:

```bash
vendor/bin/phpunit --testdox
```

# 8. Összegzés

Ebben a leckében megismerkedtünk:

* a Laravel tesztelési rendszerével
* az **in-memory SQLite adatbázissal**
* a `RefreshDatabase` trait használatával
* a **Factory osztályokkal**
* a **Feature tesztek és Unit tesztek különbségével**
* CRUD műveletek tesztelésével
* a tesztek futtatásával PHPUnit segítségével

Az automata tesztek segítenek abban, hogy az alkalmazás működése **későbbi módosítások után is megbízható maradjon**.
